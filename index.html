构建流程

不是导入脚本，是可复现的科研数据链路

项目把“原始论文元数据”转换为可查询的图结构：全量构建负责形成知识底座，增量流程负责持续接入新增论文。 这套流程同时保留源码、运行脚本、统计报告和验收文档，适合课题组后续复用。

元数据接入

读取十万级论文元数据，兼容字段缺失、来源不统一和大 JSON 流式解析。

实体清洗与匹配

清洗标题、作者、会议、年份、关键词，抽取 DOI、arXiv ID、DBLP key 并进行多标识去重。

图谱建模

定义论文、作者、会议、年份、关键词、主题等实体及 AUTHORED_BY、PUBLISHED_IN 等关系。

Neo4j 验证

生成节点和关系 CSV，执行约束、导入、检查脚本，完成本地大规模入库验证。

增量合入

新增论文通过 JSON/JSONL 输入生成 MERGE 脚本，避免重复建点和全量重跑。

图谱结构

以 Paper 为中心的科研文献结构

图谱结构围绕论文节点展开，把作者、会议、年份、关键词和主题连接成可分析的网络。 后续可以继续接入引用边、语义相似关系、主题层级和科研问答模块。

Paper

标题、摘要、DOI、arXiv、DBLP key、PDF URL、年份、会议等属性。

Author

通过 AUTHORED_BY 连接论文，并保留作者顺序。

Venue

覆盖 INTERSPEECH、NeurIPS、AAAI、CVPR、ACL、EMNLP 等来源。

Year

支持按年份统计、筛选和研究趋势分析，当前数据覆盖 2004-2026。

关键词 / 主题

用于主题组织、热点聚合和跨会议关键词分布分析。

引用关系扩展

已实现 DBLP XML 引用边抽取与诊断，为接入 OpenAlex / Semantic Scholar 预留扩展。

GraphRAG Lab

从图谱可视化到图增强检索

这一原型复现 GraphRAG 的核心思想：先从论文文本中召回候选，再利用作者、关键词、会议和年份关系扩展邻域， 最后把结果和证据路径一起返回。它不是替代完整后端，而是验证图增强检索能否接到现有 Neo4j 知识底座上。

检索模式
文本召回

从标题、摘要、关键词和会议信息中召回候选论文，作为普通 RAG 的对照。

图邻域增强

沿 HAS_KEYWORD、AUTHORED_BY、PUBLISHED_IN、PUBLISHED_YEAR 扩展邻域，补充结构信号。

证据路径

返回论文节点及其关键词、作者、会议路径，便于解释“为什么相关”。

后续扩展

从知识图谱继续走向检索、推荐与科研问答

当前版本完成了论文元数据到 Neo4j 图谱的基础设施。后续可以继续接入更丰富的引用数据源、 语义向量检索和图谱问答模块，让这个知识底座从“可组织”进一步走向“可推理、可检索、可服务”。

可继续扩展的方向
  • 接入 OpenAlex、Semantic Scholar、Crossref 或 OpenCitations，补充论文间引用关系。
  • 在关键词与摘要基础上加入语义向量检索，支持主题相近论文发现。
  • 将 Neo4j 图查询与轻量 RAG 结合，探索面向科研文献的问答原型。
