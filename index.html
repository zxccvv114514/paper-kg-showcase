Pipeline

从原始论文数据到科研知识底座

这个项目不是简单的数据导入脚本，而是一个可复现、可验收、可扩展的科研数据工程流程。 全量构建用于形成基础图谱，增量流程用于后续新增论文的持续合入。

01

Metadata Ingest

读取十万级论文元数据，兼容字段缺失、来源不统一和大 JSON 流式解析。

02

Normalize

清洗标题、作者、关键词、年份和来源字段，抽取 DOI、arXiv ID、DBLP key。

03

Graph Model

定义论文、作者、会议、年份、关键词、主题等实体及多维关系。

04

Neo4j Load

生成节点与关系 CSV，执行约束、导入、检查脚本，完成本地入库验收。

05

Incremental Merge

新增论文通过多标识匹配生成 MERGE 脚本，避免重复建点和全量重跑。

Schema

围绕科研文献组织的图谱结构

Schema 以“论文”为中心，把作者、会议、年份、关键词和主题组织为可查询的图结构。 后续可继续接入引用数据、语义相似关系、主题层级和检索问答模块。

Paper

标题、摘要、DOI、arXiv、DBLP key、PDF URL、年份、会议等属性。

Author

通过 AUTHORED_BY 关系连接论文，并保留作者顺序信息。

Venue

覆盖 INTERSPEECH、NeurIPS、AAAI、CVPR、ACL、EMNLP、ICLR 等来源。

Year

支持按年份统计、检索和趋势分析，当前数据覆盖 2004-2026。

Keyword / Subject

用于文献主题组织、研究热点聚合和跨会议关键词分布分析。

CITES Extension

已实现 DBLP XML 引用边抽取与诊断逻辑，为后续接入更合适引用源预留扩展。
